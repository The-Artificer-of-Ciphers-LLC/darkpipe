Output: Go packages under dns/provider/ with Cloudflare and Route53 implementations, provider auto-detection, dry-run wrapper, and propagation checker.

  <action>
    Create the dns/provider/ package with the provider abstraction layer:

    **interface.go** - DNSProvider interface (per user decision: Go interface for community extensibility):
    - `type DNSProvider interface` with methods:
      - `CreateRecord(ctx context.Context, rec Record) error`
      - `UpdateRecord(ctx context.Context, recordID string, rec Record) error`
      - `ListRecords(ctx context.Context, filter RecordFilter) ([]Record, error)`
      - `DeleteRecord(ctx context.Context, recordID string) error`
      - `GetZoneID(ctx context.Context, domain string) (string, error)`
      - `Name() string` - returns provider name ("cloudflare", "route53", "unknown")
    - `type Record struct { ID, Type, Name, Content string; TTL int; Priority *int }`
    - `type RecordFilter struct { Type, Name string }`

    **dryrun.go** - DryRunProvider wrapper (per user decision: dry-run by default, require --apply):
    - `type DryRunProvider struct` wraps underlying DNSProvider.
    - `NewDryRunProvider(underlying DNSProvider, dryRun bool) *DryRunProvider`
    - Write operations (CreateRecord, UpdateRecord, DeleteRecord) print what WOULD happen in dry-run mode and return nil. Format: `[DRY RUN] Would create {Type} record: {Name} -> {Content}`
    - Read operations (ListRecords, GetZoneID) always pass through to underlying provider (reads are safe).
    - `IsDryRun() bool` - allows callers to check mode.

    **detector.go** - Auto-detect DNS provider via NS records (per user decision: no need for user to specify provider):
    - `DetectProvider(ctx context.Context, domain string) (string, error)` - queries NS records for domain using miekg/dns against 8.8.8.8:53. Match NS hostnames: contains "cloudflare.com" -> "cloudflare", contains "awsdns" -> "route53", otherwise -> "unknown".
    - `NewProviderFromDetection(ctx context.Context, domain string, cfg *config.DNSConfig) (DNSProvider, error)` - detects provider, creates appropriate client from environment variables (CLOUDFLARE_API_TOKEN for Cloudflare, AWS_ACCESS_KEY_ID + AWS_SECRET_ACCESS_KEY for Route53). Returns nil provider with descriptive message for "unknown" (falls back to manual guide).

    **propagation.go** - DNS propagation polling (per user decision: built-in propagation wait with timeout):
    - `WaitForPropagation(ctx context.Context, name string, recordType string, expectedValue string, timeout time.Duration, servers []string) error` - polls DNS servers every 5 seconds until ALL servers return the expected value or timeout. Default servers: 8.8.8.8:53 (Google), 1.1.1.1:53 (Cloudflare), 208.67.222.222:53 (OpenDNS). Print progress: "Waiting for DNS propagation... ({N}/{total} servers confirmed)". Use miekg/dns for queries (not stdlib net, per research: need controlled DNS server selection).
    - Support record types: TXT, MX, A, CNAME.
    - Default timeout 5 minutes (configurable via DNS_PROPAGATION_TIMEOUT env var).

    Add dependency: `go get github.com/miekg/dns`

    Tests should cover:
    - DryRunProvider intercepts writes but passes reads
    - DryRunProvider in apply mode passes all operations through
    - NS-based detection returns correct provider strings (mock DNS responses)
    - Propagation polling succeeds when all servers confirm
    - Propagation polling times out when servers don't confirm
    - Unknown provider detection returns "unknown" not error

    Implement Cloudflare and Route53 DNS provider clients:

    **cloudflare/client.go** - Cloudflare implementation using cloudflare-go v6 (per research: Stainless-generated SDK):
    - `type Client struct` wraps cloudflare-go v6 client.
    - `NewClient(apiToken string) (*Client, error)` - creates client using `cloudflare.NewClient(option.WithAPIToken(apiToken))`. API token from CLOUDFLARE_API_TOKEN env var (per user decision: env vars only, 12-factor).
    - `GetZoneID(ctx, domain)` - uses `client.Zones.List()` with name filter to find zone.
    - `CreateRecord(ctx, rec)` - uses `client.DNS.Records.New()` with zone ID. Handle TXT records (SPF, DKIM, DMARC) and MX records. Set TTL from record (default 3600).
    - `UpdateRecord(ctx, recordID, rec)` - uses `client.DNS.Records.Update()`.
    - `ListRecords(ctx, filter)` - uses `client.DNS.Records.List()` with type and name filters. Convert Cloudflare records to provider.Record.
    - `DeleteRecord(ctx, recordID)` - uses `client.DNS.Records.Delete()`.
    - `Name()` returns "cloudflare".
    - Before creating SPF record, check for existing SPF TXT records and update instead of creating duplicate (per pitfall #8: multiple SPF records break authentication).

    Add dependency: `go get github.com/cloudflare/cloudflare-go/v6`

    **route53/client.go** - AWS Route53 implementation using aws-sdk-go-v2:
    - `type Client struct` wraps route53 service client.
    - `NewClient(ctx context.Context) (*Client, error)` - creates client using `config.LoadDefaultConfig(ctx)` then `route53.NewFromConfig(cfg)`. Credentials from AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY, AWS_REGION env vars (standard AWS SDK credential chain).
    - `GetZoneID(ctx, domain)` - uses `ListHostedZonesByName` to find hosted zone for domain. Strip leading "/" from hosted zone ID for use in other calls.
    - `CreateRecord(ctx, rec)` - uses `ChangeResourceRecordSets` with `ChangeActionCreate`. Route53 requires TXT record values wrapped in extra quotes (`"\"value\""`). Handle MX records with priority in value field.
    - `UpdateRecord(ctx, recordID, rec)` - uses `ChangeResourceRecordSets` with `ChangeActionUpsert` (Route53 doesn't use record IDs, uses name+type for identification).
    - `ListRecords(ctx, filter)` - uses `ListResourceRecordSets` with HostedZoneId. Filter by type and name on client side.
    - `DeleteRecord(ctx, recordID)` - uses `ChangeResourceRecordSets` with `ChangeActionDelete` (requires full record details, not just ID).
    - `Name()` returns "route53".
    - Before creating SPF record, list existing TXT records for domain and UPSERT instead of CREATE if SPF already exists (per pitfall #8).

    Add dependencies:

    go get github.com/aws/aws-sdk-go-v2/config
    go get github.com/aws/aws-sdk-go-v2/service/route53

    - Route53: test hosted zone ID parsing (strip leading /)
    - Route53: test UPSERT for existing SPF records
    - Both: verify they implement DNSProvider interface at compile time (`var _ provider.DNSProvider = (*Client)(nil)`)
