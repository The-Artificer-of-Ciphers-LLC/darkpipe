    **1. Platform deployment guides (deploy/platform-guides/):**

    Each guide follows the same structure:
    - Prerequisites (hardware, OS version, Docker version)
    - Quick start (3-5 commands)
    - Detailed steps
    - Platform-specific notes/limitations
    - Troubleshooting

    **raspberry-pi.md:**
    - Prerequisites: RPi4 4GB+ recommended (2GB possible with Maddy), Raspberry Pi OS 64-bit or Ubuntu Server 24.04, Docker 27+
    - Recommend USB3 SSD boot (not SD card) for mail storage performance
    - Memory optimization: if 2GB model, recommend Maddy (lighter than Stalwart), disable webmail, or use SnappyMail (128MB) over Roundcube (256MB)
    - Setup script will detect available RAM and warn if <3GB
    - Swap configuration: add 2GB swap on SSD (NOT SD card), `sudo fallocate -l 2G /swapfile`
    - Docker install: `curl -fsSL https://get.docker.com | sh`
    - Quick start: download darkpipe-setup-linux-arm64, run setup, docker compose up
    - Troubleshooting: OOM killer (check `dmesg | grep oom`), SD card wear, thermal throttling

    **truenas-scale.md:**
    - Prerequisites: TrueNAS Scale 24.10+ (Electric Eel) for Docker Compose support, earlier versions use Kubernetes (different workflow)
    - Method 1: Custom App via UI (use templates from deploy/templates/truenas-scale/)
    - Method 2: Docker Compose via SSH (for advanced users)
    - Storage: Create dedicated dataset for DarkPipe (e.g., /mnt/pool/apps/darkpipe)
    - Networking: Host networking recommended for port 25 (container networking may have issues with SMTP)
    - Note: Docker secrets may not work via TrueNAS UI -- use environment variables with password masking
    - For TrueNAS <24.10 (Kubernetes-based): Not directly supported; recommend upgrading to 24.10+

    **unraid.md:**
    - Prerequisites: Unraid 6.12+, Docker enabled
    - Method 1: Community Applications (install DarkPipe from CA, XML template)
    - Method 2: Docker Compose plugin (Compose Manager plugin for full stack)
    - Storage: /mnt/user/appdata/darkpipe/ for persistent data
    - Network: Bridge mode for most users, host mode if bridge has issues with SMTP
    - Note: XML template covers cloud-relay only; full stack requires Docker Compose

    **proxmox-lxc.md:**
    - Prerequisites: Proxmox VE 8.x, privileged or unprivileged LXC container
    - Create Debian/Ubuntu LXC container with 2GB+ RAM, 20GB+ disk
    - Install Docker inside LXC (requires nesting=1 feature for unprivileged containers)
    - Quick start: same as generic Docker (download setup, run, compose up)
    - Note: WireGuard in LXC requires privileged container OR passing /dev/net/tun

    **synology-nas.md:**
    - Prerequisites: Synology DSM 7.2+, Container Manager package installed
    - DSM 7.2+ supports Docker Compose natively through Container Manager
    - Upload docker-compose.yml via Container Manager Project feature
    - Storage: Create shared folder for DarkPipe data
    - Networking: Use bridge network, map port 25 explicitly
    - Note: Synology's Container Manager may not support Docker secrets -- use environment variables

    **mac-silicon.md:**
    - Prerequisites: macOS 14+ (Sonoma), Docker Desktop or OrbStack, Apple Silicon (M1/M2/M3/M4)
    - Primary use: Development/testing (not production mail server)
    - Images are arm64 native -- no Rosetta translation needed
    - Quick start: install Docker Desktop/OrbStack, clone repo, run darkpipe-setup-darwin-arm64, docker compose up
    - Note: Port 25 is blocked on macOS by default -- for testing only, use alternate port or deploy cloud-relay on a VPS
