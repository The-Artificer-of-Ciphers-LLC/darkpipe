       - Struct `HomeConfig` with fields: PrivateKey, Address (default "10.8.0.2/24"), CloudPubKey, CloudEndpoint, CloudAllowedIPs (default "10.8.0.1/32"), PersistentKeepalive (default 25).
       - Function `GenerateCloudConfig(cfg CloudConfig) (string, error)` that renders cloud WireGuard conf using text/template. Output must be valid wg-quick format.
       - Function `GenerateHomeConfig(cfg HomeConfig) (string, error)` that renders home WireGuard conf. MUST include `PersistentKeepalive = 25` in the Peer section -- this is critical for NAT traversal without port forwarding.
       - Function `WriteConfig(content string, path string) error` that writes config to file with 0600 permissions (private key security).

<task type="auto">
  <name>Task 2: Create WireGuard deployment scripts with systemd auto-restart</name>
  <files>
    deploy/wireguard/cloud-setup.sh
    deploy/wireguard/home-setup.sh
    deploy/wireguard/systemd/wg-quick@wg0.service.d/override.conf
  </files>
  <action>
    1. Create `deploy/wireguard/cloud-setup.sh` (bash, executable):
       - Shebang: `#!/usr/bin/env bash` with `set -euo pipefail`.
       - Check for root (required for WireGuard setup).
       - Install wireguard and wireguard-tools via apt (Debian/Ubuntu -- document that other distros need manual install).
       - Verify kernel module loads: `modprobe wireguard`.
       - Accept config file path as argument (generated by Go tooling), copy to `/etc/wireguard/wg0.conf` with mode 0600.
       - Enable and start systemd service: `systemctl enable wg-quick@wg0 && systemctl start wg-quick@wg0`.
       - Apply systemd override (from override.conf) for Restart=on-failure.
       - Enable IP forwarding: `sysctl -w net.ipv4.ip_forward=1` and persist in `/etc/sysctl.d/99-wireguard.conf`.
       - Open UDP port 51820 in ufw if ufw is active (skip if not installed).
       - Print summary: interface status via `wg show wg0`.

    2. Create `deploy/wireguard/home-setup.sh` (bash, executable):
       - Same shebang and safety flags.
       - Install wireguard and wireguard-tools via apt.
       - Verify kernel module loads.
       - Accept config file path as argument, copy to `/etc/wireguard/wg0.conf` with mode 0600.
       - Apply systemd override for Restart=on-failure with RestartSec=30.
       - Enable and start: `systemctl enable wg-quick@wg0 && systemctl start wg-quick@wg0`.
       - NO firewall changes needed on home device (NAT traversal is outbound-only from home).
       - NO IP forwarding needed on home device.
       - Print summary: interface status, verify PersistentKeepalive is active.

    3. Create `deploy/wireguard/systemd/wg-quick@wg0.service.d/override.conf`:
       ```ini
       [Service]
       Restart=on-failure
       RestartSec=30
       ```
       This ensures WireGuard auto-restarts after failures. The 30s delay prevents rapid restart loops.

    4. Make both scripts executable: `chmod +x deploy/wireguard/*.sh`.
  </action>
  <verify>
    Run `bash -n deploy/wireguard/cloud-setup.sh` and `bash -n deploy/wireguard/home-setup.sh` -- both pass syntax check. Verify scripts are executable (`test -x`). Verify override.conf contains Restart=on-failure.
  </verify>
  <done>
    Deployment scripts for cloud and home WireGuard setup exist, pass syntax validation, are executable, and include systemd auto-restart configuration. Scripts handle package installation, config deployment, firewall rules (cloud only), and IP forwarding (cloud only).
  </done>
